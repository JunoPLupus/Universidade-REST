API REST para gestão de cursos, disciplinas, professores e lecionamentos, com autenticação JWT. Construída com Node.js, TypeScript, Express e PostgreSQL (via Prisma ORM).

== Pré-requisitos

* Node.js 24+
* PostgreSQL rodando localmente (ou via Docker)

== Configuração do ambiente

Copie o arquivo de exemplo e preencha as variáveis:

[source,bash]
----
cp .env.example .env
----

[cols="1,2", options="header"]
|===
| Variável | Descrição

| `PORT`
| Porta da API (padrão: `3000`)

| `DATABASE_URL`
| String de conexão PostgreSQL para o banco principal

| `DATABASE_URL_TEST`
| String de conexão PostgreSQL para o banco de testes (tabelas limpas a cada execução)

| `JWT_SECRET`
| Segredo para assinar tokens JWT

| `JWT_EXPIRES_IN`
| Tempo de expiração dos tokens (ex.: `2h`)
|===

== Rodando localmente

[source,bash]
----
# Instalar dependências
npm install

# Aplicar as migrations no banco principal (cria as tabelas e gera o Prisma Client)
npx prisma migrate deploy
npx prisma generate

# Iniciar em modo desenvolvimento (hot reload)
npm run dev

# Ou compilar e iniciar em produção
npm run build
npm start
----

A API estará disponível em http://localhost:3000. +
A documentação Swagger estará em http://localhost:3000/docs.

=== Banco de testes

O banco de testes é controlado pela variável `DATABASE_URL_TEST` no `.env`. Como ele não possui histórico de migrations, use `prisma db push` para criar as tabelas:

[source,bash]
----
# Linux / macOS
DATABASE_URL="<valor de DATABASE_URL_TEST>" npx prisma db push

# Windows (PowerShell)
$env:DATABASE_URL="<valor de DATABASE_URL_TEST>"; npx prisma db push
----

Substitua `<valor de DATABASE_URL_TEST>` pela string de conexão do banco de testes (ex.: `postgresql://postgres:senha@localhost:5432/universidade_test?schema=public`).

NOTE: O banco de testes é limpo automaticamente antes de cada execução dos testes de integração. Não é necessário populá-lo manualmente.

== Rodando com Docker

[source,bash]
----
# Build da imagem
docker build -t universidade-rest .

# Rodar o container
docker run -p 3000:3000 -e DATABASE_URL="postgresql://usuario:senha@host.docker.internal:5432/universidade?schema=public" \
  -e JWT_SECRET="seu-segredo" -e JWT_EXPIRES_IN="2h" universidade-rest
----

NOTE: O container não inclui o PostgreSQL. Use um banco externo ou suba um via `docker run postgres`.

== Testes

Antes de rodar os testes de integração pela primeira vez (ou após clonar o repositório em outra máquina), configure o banco de testes conforme descrito em <<banco-de-testes, Banco de testes>>.

[source,bash]
----
# Testes unitários
npm run test:unit

# Testes de integração (requer DATABASE_URL_TEST configurada e banco de testes criado)
npm run test:integration

# Todos os testes
npm test
----

== Autenticação

A API utiliza JWT (Bearer Token). Obtenha um token via `POST /auth/login` e inclua-o no cabeçalho de cada requisição protegida:

----
Authorization: Bearer <token>
----

Níveis de acesso:

[cols="1,3", options="header"]
|===
| Nível | Permissões

| *Público*
| Acesso irrestrito — qualquer cliente pode consultar

| *Autenticado*
| Requer token JWT válido

| *Admin*
| Requer token JWT com `role: ADMIN`
|===

== Endpoints

=== Auth

[cols="1,2,2,1", options="header"]
|===
| Método | Rota | Descrição | Acesso

| `POST`
| `/auth/login`
| Autenticar e obter token JWT
| Público
|===

=== Professores

[cols="1,2,2,1", options="header"]
|===
| Método | Rota | Descrição | Acesso

| `GET`
| `/professores`
| Listar professores (filtros opcionais) — retorna sempre a visão pública `{ matricula, nome }`
| Público

| `GET`
| `/professores/:mat`
| Buscar professor por matrícula — retorna visão completa para admin ou para o próprio professor; visão pública `{ matricula, nome }` para demais casos
| Público (resposta condicional)

| `POST`
| `/professores`
| Cadastrar professor
| Admin

| `PATCH`
| `/professores/:mat`
| Editar professor (admin edita qualquer; professor edita o próprio)
| Autenticado

| `DELETE`
| `/professores/:mat`
| Excluir professor
| Admin
|===

=== Cursos

[cols="1,2,2,1", options="header"]
|===
| Método | Rota | Descrição | Acesso

| `GET`
| `/cursos`
| Listar cursos (filtros opcionais por nome e código)
| Público

| `GET`
| `/cursos/:codigo`
| Buscar curso por código
| Público

| `POST`
| `/cursos`
| Cadastrar curso
| Admin

| `PATCH`
| `/cursos/:codigo`
| Editar curso (campos opcionais)
| Admin

| `DELETE`
| `/cursos/:codigo`
| Excluir curso
| Admin

| `DELETE`
| `/cursos/:codigo/disciplinas`
| Excluir todas as disciplinas do curso
| Admin
|===

=== Lecionamentos

[cols="1,2,2,1", options="header"]
|===
| Método | Rota | Descrição | Acesso

| `GET`
| `/lecionamentos`
| Listar lecionamentos (filtros opcionais: `codigo`, `codDisciplina`, `matProfessor`, `codSemestre`, `turno`, `diaSemana`)
| Público

| `GET`
| `/lecionamentos/:codigo`
| Buscar lecionamento por código
| Público

| `POST`
| `/lecionamentos`
| Cadastrar lecionamento — código gerado automaticamente; semestre criado automaticamente se não existir
| Admin

| `PATCH`
| `/lecionamentos/:codigo`
| Editar lecionamento (campos opcionais) — código recalculado se semestre ou disciplina forem alterados
| Admin

| `DELETE`
| `/lecionamentos/:codigo`
| Excluir lecionamento
| Admin
|===
